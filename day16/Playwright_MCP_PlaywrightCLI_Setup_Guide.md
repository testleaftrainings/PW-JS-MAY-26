# Playwright MCP Server + Playwright CLI + GitHub Copilot Agents Setup Guide

## Objective

Configure a Playwright project using: - GitHub Copilot Agents (Planner,
Generator, Healer) - Playwright MCP Server - Playwright CLI

## Architecture

``` text
You
 │
 ▼
GitHub Copilot Agent
 │
 ├── Planner
 ├── Generator
 └── Healer
 │
 ▼
Playwright MCP Server
 │
 ▼
Playwright CLI
 │
 ▼
Browser
```

## Step 1 - Create Workspace

``` bash
mkdir Playwright-MCP-Demo
cd Playwright-MCP-Demo
code .
```

## Step 2 - Initialize Playwright

``` bash
npm init playwright@latest
```

Choose: - TypeScript - Install browsers - tests folder

## Step 3 - Update Playwright

``` bash
npm install -D @playwright/test@latest
```

Verify package.json contains @playwright/test.

## Step 4 - Initialize Playwright Agents

``` bash
npx playwright init-agents --loop=vscode
```

## Step 5 - Install Playwright CLI

Global installation:

``` bash
npm install -g @playwright/cli@latest
```

Or invoke using npx.

## Step 6 - Install CLI Skills

``` bash
playwright-cli install --skills
```

## Step 7 - Configure MCP Server

Example configuration:

``` json
{
  "servers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

## Step 8 - Verify Installation

``` bash
playwright-cli --version
```

``` bash
npx playwright --version
```

## Step 9 - Planner Agent

Create an execution plan only.

``` text
# Role

You are a Senior QA Test Architect.

# Objective

Analyze the business requirement and create a detailed execution plan for automating the successful creation of a Lead in the Leaftaps CRM application.

Do NOT generate any Playwright code.

# Application Details

Application URL:
http://leaftaps.com/opentaps/control/main

Credentials:
Username: democsr2
Password: crmsfa

# Business Scenario

1. Launch the application.
2. Login using the provided credentials.
3. Navigate to CRM/SFA.
4. Navigate to the Create Lead page.
5. Populate all mandatory fields.
6. Submit the Lead creation form.
7. Verify that the Lead is created successfully.

# Expected Output

Create a structured execution plan containing:

## 1. Business Flow
- End-to-end user journey
- Navigation path

## 2. Pages Involved
Identify all application pages involved in the workflow.

## 3. Preconditions

## 4. Test Data
- Fixed test data
- Dynamic test data
- Fields suitable for Faker-generated values

## 5. Execution Steps
Step-by-step functional flow.

## 6. Validation Points
Identify all validations that should be performed.

## 7. Expected Results

## 8. Risks / Edge Cases
Identify possible validation failures and edge cases.

## 9. Recommended Project Structure
Suggest the required Page Objects and Test files.

Example:

pages/
- LoginPage
- HomePage
- CreateLeadPage

tests/
- createLead.spec.ts

# Constraints

- Do not generate automation code.
- Do not generate locators.
- Do not generate Playwright APIs.
- Do not assume implementation details.

The generated execution plan will be used as the input for the Generator Agent.
``` 
Review and approve the plan.

## Step 10 - Generator Agent

Attach the planner output.

``` text
# Role

You are a Senior Playwright Automation Engineer specializing in TypeScript and Playwright Test.

# Objective

Using the attached execution plan generated by the Planner Agent, generate a complete production-ready Playwright automation framework.

Use the Playwright MCP Server and Playwright CLI to inspect the live application whenever required to identify accurate element locators and validate the application flow.

# Implementation Requirements

Framework:
- Playwright Test

Language:
- TypeScript

Design Pattern:
- Page Object Model (POM)

Project Structure:

pages/
tests/

Generate the required Page Objects and Test files based on the Planner output.

# Coding Standards

- Follow Playwright best practices.
- Use Playwright recommended locators (getByRole, getByLabel, getByPlaceholder, getByText, etc.).
- Avoid XPath unless absolutely necessary.
- Do not use page.waitForTimeout().
- Use explicit assertions with expect().
- Keep the code modular and reusable.
- Use meaningful class names, method names, and variable names.
- Add concise comments only where necessary.
- Follow clean code principles.

# Test Data

- Use the credentials provided in the Planner output.
- Generate all remaining test data dynamically using the Faker library.
- Ensure generated values are realistic and unique where applicable.

# Browser Inspection

Use Playwright CLI through the MCP Server to:

- Launch the application.
- Inspect the live DOM.
- Identify reliable element locators.
- Verify navigation flow.
- Validate mandatory fields before generating automation.

Do not guess locators when they can be obtained from the live application.

# Framework Expectations

Generate:

- Page Objects
- Test Specification(s)
- Required imports
- Reusable methods
- Assertions
- Faker integration

Maintain a clear separation between Page Objects and Test files.

# Execution

After generating the framework:

1. Install any missing dependencies.
2. Compile the project.
3. Execute the Playwright test.
4. If compilation or runtime errors occur:
   - Analyze the failure.
   - Fix only the affected code.
   - Re-run the test.
5. Continue until the test executes successfully.

# Final Output

Provide:

- Folder structure
- Generated source code
- Execution summary
- Test result summary
- Playwright HTML Report command

Do not generate explanations or theory unless required.
Return production-ready automation only.
```

## Step 11 - Execute

``` bash
npx playwright test
```

## Step 12 - Healer Agent

Fix: - Locator issues - Assertion failures - Synchronization issues

Update only affected files.

## Step 13 - Execute Again

``` bash
npx playwright test
```

## Step 14 - Open Report

``` bash
npx playwright show-report
```

## Best Practices

-   Create the Playwright project first.
-   Install or invoke Playwright CLI separately.
-   Configure the MCP Server before using agents.
-   Planner -\> Generator -\> Healer.
-   Avoid hard waits.
-   Use Playwright recommended locators.
